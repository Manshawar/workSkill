# 输入边界值清单

> 对流程里**每一个输入框**逐行套用本清单。Step 3 边界值组生成时加载。

## 用法

拿到一个输入框先问三个问题，再按下表生成用例：

1. 它是什么 `type`？（text / tel / number / textarea / …）
2. 有没有 `maxlength`？
3. 有没有格式约束？（手机号位数、验证码位数、邮箱格式…）

## 边界值清单

| # | 边界情形 | 适用类型 | 期望 |
| --- | --- | --- | --- |
| BV-1 | 空值（不填直接提交） | 全部 | 必填校验拦截 / 默认值生效 |
| BV-2 | 1 个字符 | 文本 / 文本域 | 接受或按最小长度报错 |
| BV-3 | 最大长度（如 20 字） | 有 maxlength | 接受 |
| BV-4 | 超长（最大 +1） | 有 maxlength | 截断 / 拒绝输入 |
| BV-5 | 纯空格 / 前后空格 | 文本 | 是否 trim、是否算空 |
| BV-6 | 特殊字符：emoji / `<script>` / 引号 / 换行 | 文本 / 文本域 | 不崩、正确转义 |
| BV-7 | SQL / XSS 样字符串 | 文本 | 仅前端表现层不渲染执行 |
| BV-8 | 格式错误（手机号少 1 位、验证码非 6 位） | tel / 格式约束 | 前端格式校验拦截 |
| BV-9 | 0 / 负数 / 小数 | 数值型（排序等） | 按业务边界接受或拒 |
| BV-10 | 超上限（如排序填 99999999） | 数值型 | 不溢出、不撑爆布局 |
| BV-11 | 粘贴超长文本 | 文本 / 文本域 | 同 BV-4 |
| BV-12 | 只读框强制改值 | readonly input | 改不动 / 提交仍用原值 |

## 最小要求

**每个输入框至少出 3 条边界值用例**，优先级：空 → 最大长度 → 超长。每条都要带「注释」。

边界值组专攻异常与临界——**happy path（正常值）留给 E2E 组**，不要在边界值组里堆正常值凑数。
